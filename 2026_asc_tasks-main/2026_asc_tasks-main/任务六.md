# 视觉组任务六：YOLO 目标检测与实战落地

**前言：** 在经历了传统视觉特征提取（任务四）和深度学习基础（任务五）后，我们正式迈入目前工业界和各类机器人/智能车比赛中最流行的目标检测算法——**YOLO (You Only Look Once)**。YOLO 的核心优势在于“快”和“准”，非常适合需要在边缘计算设备上实时运行的场景。

本次任务将带领大家从“调用预训练模型”到“训练自己的数据集”，最后完成一个“结合业务逻辑的实际应用”。

## 阶段一：站在巨人的肩膀上 —— 预训练模型推理

YOLO 官方提供了在 COCO 数据集上训练好的权重文件（`.pt`），能够识别 80 种常见的物体（如人、车、猫、狗等）。

* **任务要求**：
    1. **环境准备**：克隆 [YOLOv5 的官方 GitHub 仓库](https://github.com/ultralytics/yolov5) 并配置环境；同时安装 YOLOv8 的库（`pip install ultralytics`）。
    2. **YOLOv5 推理**：下载官方的 `yolov5s.pt` 权重，使用 `detect.py` 脚本，对经典的测试图 `bus.jpg` 和 `zidane.jpg`（通常在 `data/images` 目录下）进行预测。
    3. **YOLOv8 推理**：编写简单的 Python 脚本或使用命令行（CLI），调用 `yolov8n.pt` 预测同样的图片。
    * ***预期效果***：输出带有各类物体识别框（Bounding Box）、置信度（Confidence）和类别名称的图片。

## 阶段二：炼丹实战 —— 自定义数据集训练

在实际比赛中，我们需要识别的往往是特定的目标（例如赛道上的锥桶、特定的引导牌等），这就需要用我们自己的数据来“微调”模型。

* **任务要求**：
    1. **数据集准备与划分**：学长会提供一个包含自定义目标（终点绿色指示牌）的数据集（见任务六图片附件）。你需要编写一个 Python 脚本（利用 `os`, `random`, `shutil` 等库），将数据集的图片和 `txt` 标签按比例（例如 8:1:1）划分为**训练集 (train)**、**验证集 (val)** 和 **测试集 (test)**。
    2. **配置文件编写**：编写对应的 `data.yaml` 文件，指明 train/val 的路径、类别数量（`nc`）和类别名称（`names`）。
    3. **模型训练**：
        * 使用 YOLOv5 训练你的数据集，生成自有的权重文件。
        * 使用 YOLOv8 训练你的数据集，同样生成自有的权重文件。
    * ***预期效果***：在训练结束后，查看 `runs/train` 目录下的结果，了解 Precision、Recall、mAP50 等评价指标的含义，并保留你训练出的最优权重 `best.pt`。

## 阶段三：视觉与逻辑的结合 —— 终点线判定应用

模型训练出来只是第一步，如何将其转化为智能车的控制信号才是关键。在这一步，我们将结合你训练出的模型和真实的业务逻辑。

* **背景**：智能车正驶向终点的绿色指示牌（见测试视频或图片附件）。我们需要利用视觉算法判断车辆是否“越过终点线”。
* **任务要求**：
    1. **目标提取**：编写 Python 脚本，使用你自己在阶段二训练好的 `.pt` 权重文件，针对测试集里的图片，识别画面中的“绿色方框”。
    2. **几何逻辑判定**：
        * 获取识别框的坐标数据 `(x_min, y_min, x_max, y_max)`。
        * 在画面高度的 **3/4 处**（即 `Y = 图像总高度 * 0.75`）画一条红色的水平基准线。
        * **判定条件**：当识别到的绿色方框的**下边缘**（即 `y_max`）碰到或越过这条基准线时，在画面上用大字体打印 **"到达终点 / Finish!"**。
    * ***预期效果***：画面中有一个跟着绿色方框移动的检测框（附带置信度），画面下方 3/4 处有一条红线。当绿色方框随着车辆靠近而在画面中变大并下沉，其底边触碰红线时，触发终点信号。
